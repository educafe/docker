===============================================================
원격 호스트 서버에 docker-machine 생성하기
===============================================================
현재까지는 모든 것을 root 계정에서 수행해야 함 (educafe 계정으로도 됨 ㅎㅎㅎ)
------------------------------------------------------------------
서버가 ssh 통신을 개인키/공개키로 수행하여야 하므로, 개인키/공개키를 생성하여 교환하여야 함
---------------------------------------------------------------------------------
root 계정으로의 ssh 로그인이 허용되어야 함 /etc/ssh/sshd_config 파일에서 수정 -> RootLogin을 yes 로 변경
--------------------------------------------------------------------------------------------
2004 머신에서
docker-machine create --driver generic \
--generic-ip-address 192.168.56.104(ubuntu1804) \
--generic-ssh-key ~/.ssh/id_rsa \
--generic-ssh-user root \
--generic-ssh-port 22 \ (선택)
node-1
과 같이 명령을 입력하면 원격 1804 서버에 docker machine을 설치되며 이후에 swarm으로 가입할 수 있다
이후 docker-machine ls 명령으로 확인하면 node-1 이름으로 docker machine이 네트워크로 연결되어 있는 것을 
확인할 수 있다.
---------------------------------------------------------------------------------
docker-machine create --driver virtualbox 명령어는 자신의 서버(노드)의 virtualbox에 새로운 
새로운 docker-machine을 생성하는 것이며, 이 때는 리눅스 내에 virtualbox가 설치되어 있어야 한다.
